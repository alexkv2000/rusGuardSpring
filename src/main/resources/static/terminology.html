Терминология проекта RusGuard

Данный документ содержит описание ключевых терминов и понятий, используемых в проекте RusGuard - системе контроля и управления доступом.

Основные термины

Сотрудник (Employee)
Пользователь системы контроля доступа, имеющий уникальный идентификатор и набор прав доступа. Сотрудник может быть связан с группами, иметь уровни доступа и проходить через точки контроля.
Пример: Иванов Иван Иванович, табельный номер 12345
Уровень доступа (Access Level)
Набор прав, определяющих возможности сотрудника в системе. Каждый уровень доступа имеет уникальный идентификатор и может быть назначен конкретным сотрудникам или группам.
Пример: "Архив", "Доступ в архивный отдел"
Группа сотрудников (Employee Group)
Иерархическая структура, объединяющая сотрудников по подразделениям или функциональным областям. Группы могут наследовать уровни доступа от родительских групп.
Пример: "Отдел продаж", "IT-отдел"
Табельный номер (Tabel Number)
Уникальный числовой идентификатор сотрудника, используемый для идентификации в системе.
Пример: 12345
Должность (Position)
Функциональная роль сотрудника в организации, определяющая его обязанности и права доступа.
Пример: "Главный инженер", "Специалист по безопасности"
Проход (Passage)
Событие, фиксирующее попытку доступа сотрудника к какой-либо точке контроля. Включает информацию о времени, месте и результате попытки.
Пример: "Проход сотрудника Иванова И.И. через дверь №100 в 10:30:15"
Точка доступа (Access Point)
Физическое устройство или логический элемент системы контроля доступа, обеспечивающий контроль прохождения сотрудников.
Пример: "Дверь входа в офис", "Контрольный пункт"
UUID (Universally Unique Identifier)
Уникальный идентификатор, используемый для однозначной идентификации объектов в системе. Используется для идентификации сотрудников, уровней доступа и других сущностей.
Пример: "a38abfd9-d277-43fb-b719-618c7c91e7a1"
Блокировка (Lock)
Состояние, при котором сотрудник временно либо постоянно лишается прав доступа к системе или определенным точкам.
Пример: "Сотрудник заблокирован", "Сотрудник разблокирован"
Электронная почта (Email)
Контактный адрес сотрудника, используемый для уведомлений и коммуникации внутри системы.
Пример: "ivanov@example.com"

Технические термины

DTO (Data Transfer Object)
Объект передачи данных, используемый для передачи информации между слоями приложения. В проекте используются DTO для передачи данных сотрудников и уровней доступа.
Пример: SaveAcsEmployeeRequest, AccessLevelIdsRequest
REST API
Архитектурный стиль взаимодействия между компонентами программного обеспечения, используемый в проекте для предоставления интерфейсов для работы с данными сотрудников и уровнями доступа.
Пример: /api/rusguard/Employee/getById, /api/rusguard/Employee/saveAcsEmployee
Spring Boot
Фреймворк для создания веб-приложений на Java, используемый в проекте для реализации REST API и бизнес-логики.
Пример: @RestController, @RequestMapping
Swagger/OpenAPI
Стандарт описания REST API, используемый в проекте для автоматической генерации документации API.
Пример: @Operation, @ApiResponses, @Schema
Лицензия (License)
Информация о лицензионном соглашении, ограничивающем использование системы. Содержит информацию о компании, номере лицензии, дате выдачи и окончания действия.
Пример: ITSNN-RUSGUARD-CORP-20260130-100
Система контроля и управления доступом (ACS)
Интегрированная система, обеспечивающая контроль доступа к объектам и ресурсам организации.
Пример: RusGuard - название системы
Паспортные данные (Passport Data)
Информация о паспорте сотрудника, включая серию и номер.
Пример: Серия 7710, номер 1010
Адрес регистрации (Registration Address)
Юридический адрес регистрации сотрудника.
Пример: "г. Москва, ул. Проспект Мира, д.1, кв.8"
Адрес проживания (Residential Address)
Адрес постоянного проживания сотрудника.
Пример: "г. Москва, ул. Проспект Мира, д.1, кв.8"
Комментарий (Comment)
Дополнительная информация о сотруднике, которая может быть полезна для административных целей.
Пример: "Сотрудник работает удаленно"
Сервис (Service)
Компонент приложения, реализующий бизнес-логику и взаимодействующий с базой данных и другими компонентами.
Пример: EmployeeService, LicenseService
Контроллер (Controller)
Компонент приложения, обрабатывающий HTTP-запросы и возвращающий ответы клиенту.
Пример: EmployeeController, LicenseControllerSwagger
Метод (Method)
Функция или процедура, реализующая определенную функциональность в системе.
Пример: getEmployeeById, setAccessLevelEmployee
Параметр (Parameter)
Значение, передаваемое в метод или функцию для выполнения определенной операции.
Пример: idEmployee, flagLock
Ответ (Response)
Результат выполнения операции, возвращаемый клиенту после обработки запроса.
Пример: JSON-объект с результатом операции
