# Инструкция по сборке проекта

## Требования

- Java 11 или выше
- Apache Maven 3.6.0 или выше
- Windows (скрипты написаны для Windows)

## Структура проекта

- `src/main/java/` - исходный код приложения
- `src/main/java/com/rusguard/client/` - классы для взаимодействия с SOAP-сервисами
- `target/generated-sources/cxf/` - автоматически сгенерированные классы для работы с SOAP-сервисами
- `src/main/lib/` - исходные Java-классы библиотеки RusGuard
- `target/lib/` - место, куда помещается JAR-файл библиотеки после выполнения скрипта
- `pom.xml` - конфигурация Maven
- `create_jar.bat` - скрипт для создания JAR-файла из библиотеки
- `build_and_run.bat` - скрипт для сборки и запуска приложения

## Процесс сборки

1. **Создание JAR-файла библиотеки**

   Сначала нужно создать JAR-файл из исходных Java-классов, расположенных в `src/main/lib/`. 
   
   Запустите скрипт:
   ```
   create_jar.bat
   ```
   
   Этот скрипт создаст файл `entities.jar` и поместит его в директорию `target/lib/`.

2. **Сборка проекта**

   После создания JAR-файла можно выполнить сборку проекта с помощью Maven:
   ```
   mvn clean package
   ```

3. **Запуск приложения**

   После успешной сборки запустите приложение:
   ```
   java -jar target/rusguard-web-service-1.0.0.jar
   ```

## Альтернативный способ (рекомендуется)

Вы можете выполнить все действия одной командой, запустив скрипт:
```
build_and_run.bat
```

Этот скрипт автоматически:
1. Создаст JAR-файл библиотеки
2. Выполнит сборку проекта
3. Запустит приложение

## Примечания

- При каждом изменении в классах библиотеки (в директории `src/main/lib/`) нужно повторно запускать `create_jar.bat`.
- JAR-файл `entities.jar` должен находиться в директории `target/lib/` перед сборкой проекта.
- При сборке Maven использует этот JAR-файл как системную зависимость.
- Для работы с SOAP-сервисами RusGuard используется Apache CXF, который генерирует классы в `target/generated-sources/cxf/`.
- Maven автоматически добавляет сгенерированные исходники в процесс сборки.
